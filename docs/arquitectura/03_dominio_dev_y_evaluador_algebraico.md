# Dominio DEV y evaluador algebraico inicial

## Objeto
Este documento fija la arquitectura mínima del dominio DEV en el estado actual del motor.

## Arquitectura declarada
La arquitectura inicial del dominio DEV queda limitada a la capa evaluadora:

```
Ω_DEV → I_DEV → C_dev^9 → Γbar_H_DEV → A_DEV → κ₃_DEV → K₃
```

No existe todavía capa generativa activa del dominio DEV.

## Política de fase
El orden correcto del frente DEV es:

1. declaración del dominio;
2. evaluador algebraico del dominio;
3. casos canónicos y laboratorio mínimo;
4. revisión de conformidad del dominio;
5. solo después, estudio de extractor o capa generativa.

## Dictamen de la fase
La fase actual solo puede declarar como hecho:
- que el dominio DEV existe ya como dominio evaluable;
- que su célula está tipada y auditada;
- y que el frente generativo DEV sigue deliberadamente cerrado.

## No procede declarar todavía
- que existe ya un agente de desarrollo del SV;
- que puede producir artefactos de software bajo control algebraico completo;
- que la web entra ya como material tipificado del dominio DEV;
- ni que la capa generativa de desarrollo está abierta.
