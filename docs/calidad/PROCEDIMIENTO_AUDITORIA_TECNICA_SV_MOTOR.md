# Procedimiento de auditoría técnica del frente motor SV

## 1. Naturaleza

Este procedimiento es operativo y subordinado. No modifica doctrina, no redefine el lenguaje y no sustituye la compuerta de seguridad estructural.

## 2. Objeto

Regular cómo se auditan los hitos técnicos del repositorio `SV-motor`, preservando la congruencia entre documentación, implementación, laboratorio y estado público declarado.

## 3. Alcance

Se aplica a cambios que afecten a:

- núcleo algebraico local;
- extractores subordinados;
- validación geométrica;
- laboratorio reproducible;
- documentación pública del frente motor;
- sincronización con repositorios superiores.

## 4. Principios

1. primacía doctrinal;
2. proporcionalidad;
3. no duplicación doctrinal;
4. trazabilidad suficiente;
5. no burocratización;
6. adversarial obligatoria;
7. publicación sobria y no sobreactuada.

## 5. Disparadores de auditoría

Debe abrirse auditoría cuando exista:

- apertura o cierre de una fase del motor;
- cambio en extractores, compuertas, validación o laboratorio;
- modificación de README o de documentos de alcance público;
- apertura de una capa online nueva;
- sincronización con un frente superior;
- preparación de publicación o demostración pública.

## 6. Secuencia mínima

1. delimitación;
2. revisión de base;
3. clasificación entre error real, mejora opcional, deuda futura o cambio improcedente;
4. adversarial explícita;
5. veredicto;
6. cierre o traslado.

## 7. Evidencia admisible

Se consideran evidencia válida:

- árbol real del repositorio;
- código ejecutable;
- suite de pruebas;
- salidas del laboratorio;
- registros y documentación pública.

No bastan por sí solos:

- nombres de archivo;
- intención declarada;
- arquitectura dibujada sin ejecución;
- metadatos aislados.

## 8. Regla de no sobreactuación

Ningún cierre técnico se presentará como motor suficiente mientras el laboratorio mínimo reproducible, la suite ampliada y la sincronización local con sedes superiores no estén materialmente acreditados.
