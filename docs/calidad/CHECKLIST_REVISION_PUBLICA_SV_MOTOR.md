# Checklist de revisión pública — SV-motor

## Repositorio y árbol material
- [ ] La versión material auditada del repositorio es legible y completa.
- [ ] No hay residuos locales impropios.
- [ ] La estructura pública coincide con el estatuto declarado.

## Gobierno y documentación
- [ ] README veraz y sobrio.
- [ ] Alcance y no-alcance explícitos.
- [ ] Dependencias superiores declaradas correctamente.
- [ ] Criterios de parada, sincronización y avance visibles.

## Paquete técnico
- [ ] El paquete instala.
- [ ] La suite de pruebas corre sin error.
- [ ] No hay imports rotos ni estructura ficticia.
- [ ] El empaquetado coincide con lo que promete el código.

## Laboratorio
- [ ] Existe laboratorio mínimo reproducible.
- [ ] Hay pseudocódigo.
- [ ] Hay script ejecutable.
- [ ] Hay salida JSON.
- [ ] Hay breve análisis.
- [ ] La U se conserva cuando procede.

## Calidad y trazabilidad
- [ ] Existen registros mínimos de calidad.
- [ ] Hay deuda viva explícita.
- [ ] Hay sincronización con repos superiores.
- [ ] Hay registro de decisiones y adversariales.

## Coherencia superior
- [ ] No contradice el Pliego.
- [ ] No contradice FT-SV-IA/001.
- [ ] No contradice la célula de seguridad estructural.
- [ ] No sobreactúa el frente NLP.

## Dictamen
- [ ] Puede avanzar.
- [ ] Debe sincronizar antes de avanzar.
- [ ] Debe parar.
