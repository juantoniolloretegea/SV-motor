# Dependencias y prevalencia

La cadena de prevalencia aplicable al repositorio es la siguiente:

SV soberano -> doctrina -> álgebra -> lenguaje -> compuertas de seguridad -> agentes especializados -> motor después.

## Consecuencias

- Ninguna capa inferior corrige silenciosamente a una superior.
- La sede del motor no reescribe el pliego, el álgebra ni el lenguaje.
- FT-SV-IA/001 regula la frontera con IA externa y no equivale al motor nativo.
- Toda integración técnica del motor debe pasar por la compuerta de seguridad estructural.
